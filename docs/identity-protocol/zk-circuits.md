# ZK Circuits

## Overview

Three Noir circuits form the cryptographic foundation of the Civyx identity protocol. Each circuit is compiled to an UltraKeccakHonk proof and verified on-chain by a Solidity verifier contract generated by Barretenberg.

All circuits use BN254 curve arithmetic and share the same proof system parameters.

| Parameter | Value |
|---|---|
| Curve | BN254 |
| Proof system | UltraKeccakHonk |
| Noir version | 1.0.0-beta.18 |
| Barretenberg version | 3.0.0-nightly.20260102 |

---

## identity.nr

Proves knowledge of a secret whose Pedersen hash equals a registered commitment. Used in flows that require basic identity proof without linking a wallet or consuming a nullifier.

```noir
fn main(
    secret: Field,           // private — never leaves the browser
    commitment: pub Field,   // public — stored on-chain at registration
) {
    let computed = std::hash::pedersen_hash([secret]);
    assert(
        computed == commitment,
        "commitment mismatch: pedersen_hash([secret]) != commitment"
    );
}
```

| Parameter | Value |
|---|---|
| Public inputs | 1 (commitment) |
| Verifier contract | IdentityVerifier |
| Verification key hash | `0x08c3a688...` |

## wallet_link.nr

Proves that a new wallet address is being linked by someone who knows the secret behind an existing commitment. Outputs a nullifier to prevent replay.

```noir
fn main(
    secret: Field,              // private
    commitment: pub Field,      // public
    nullifier: pub Field,       // public
    wallet_address: pub Field,  // public
) {
    let computed_commitment = std::hash::pedersen_hash([secret]);
    assert(
        computed_commitment == commitment,
        "commitment mismatch: pedersen_hash([secret]) != commitment"
    );

    let computed_nullifier = std::hash::pedersen_hash([secret, wallet_address]);
    assert(
        computed_nullifier == nullifier,
        "nullifier mismatch: pedersen_hash([secret, wallet_address]) != nullifier"
    );
}
```

| Parameter | Value |
|---|---|
| Public inputs | 3 (commitment, nullifier, wallet_address) |
| Proof size | 8640 bytes |
| Verifier contract | WalletLinkVerifier |
| Verification key hash | `0x2798bb7c...` |

## nullifier.nr

General-purpose circuit for one-time proof actions. Verifies both identity ownership and honest nullifier derivation for any arbitrary action identifier. Used for task verification flows that require proof of identity and prevention of replay for a specific action.

```noir
fn main(
    secret: Field,           // private
    commitment: pub Field,   // public
    nullifier: pub Field,    // public
    action_id: pub Field,    // public — identifies the specific action
) {
    let computed_commitment = std::hash::pedersen_hash([secret]);
    assert(
        computed_commitment == commitment,
        "commitment mismatch: pedersen_hash([secret]) != commitment"
    );

    let computed_nullifier = std::hash::pedersen_hash([secret, action_id]);
    assert(
        computed_nullifier == nullifier,
        "nullifier mismatch: pedersen_hash([secret, action_id]) != nullifier"
    );
}
```

| Parameter | Value |
|---|---|
| Public inputs | 3 (commitment, nullifier, action_id) |
| Verifier contract | NullifierVerifier |

## Proof Generation

All proofs are generated entirely in the browser using the Noir JS library and Barretenberg WASM. No data is sent to any server. The process:

1. Load the circuit JSON from `/public/<circuit>.json`
2. Initialise Noir with the circuit
3. Execute the circuit with the user's inputs to generate a witness
4. Pass the witness to `UltraHonkBackend` to generate the proof
5. Submit the proof and public inputs to the on-chain verifier

Proof generation typically takes **10–30 seconds** depending on device hardware.

---

## Verifier Contracts

The Solidity verifier contracts are generated by Barretenberg directly from the compiled circuit artifacts. They contain no state, no access control, and no admin functions. Each verifier is a pure deterministic mathematical check — a function of the circuit it was compiled from — that cannot be upgraded or altered after deployment.
