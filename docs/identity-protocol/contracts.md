## IdentityRegistry

**Address:** `0x56BBC4969818d4E27Fe39983f8aDee4F3e1C5c6f`

The central contract of the identity layer. Stores all commitments, stakes, wallet-to-identity mappings, and nullifier records.

**Key functions:**

| Function | Description |
|---|---|
| `registerIdentity(bytes32 commitment)` | Register new identity with PAS stake |
| `linkWallet(bytes32 commitment, bytes proof, bytes32 nullifier)` | Link wallet via ZK proof |
| `deactivateIdentity()` | Deactivate identity and recover stake |
| `reactivateIdentity(bytes32 commitment)` | Reactivate deactivated identity |
| `getCommitment(address wallet)` | Returns commitment for a wallet |
| `verifyIdentity(address wallet)` | Returns true if wallet has active identity |
| `isNullifierUsed(bytes32 nullifier)` | Returns true if nullifier is consumed |
| `getIdentityStake(address wallet)` | Returns stake amount |

**Access control roles:**

| Role | Purpose |
|---|---|
| `DEFAULT_ADMIN_ROLE` | Grant/revoke roles, update parameters |
| `PAUSER_ROLE` | Pause and unpause |
| `OPERATOR_ROLE` | Administrative deactivation |

**Events:**

| Event | Parameters |
|---|---|
| `IdentityRegistered` | `address wallet, bytes32 commitment, uint256 stake` |
| `WalletLinked` | `address wallet, bytes32 commitment` |
| `IdentityDeactivated` | `bytes32 commitment, address by, uint256 stakeReturned` |
| `IdentityReactivated` | `bytes32 commitment, uint256 newStake` |

**OpenZeppelin:** `AccessControl`, `ReentrancyGuard`, `Pausable`

## WalletLinkVerifier

**Address:** `0x72CC5BA2958CB688B00dFE2E578Db3BbB79eD311`

Generated by Barretenberg from the compiled `wallet_link` Noir circuit. A pure verification function — no state, no access control, not upgradeable.

**Interface:**
```solidity
function verify(
    bytes calldata proof,
    bytes32[] calldata publicInputs
) external view returns (bool);
```

**Public inputs (in order):**

| Index | Name | Description |
|---|---|---|
| 0 | `commitment` | pedersen_hash([secret]) |
| 1 | `nullifier` | pedersen_hash([secret, wallet_address]) |
| 2 | `wallet_address` | Linked address padded to bytes32 |

**Proof format:** UltraKeccakHonk, 8640 bytes

## NullifierVerifier

**Address:** `0x0454a4798602babb16529F49920E8B2f4a747Bb2`

Verifier for the general-purpose nullifier circuit. Used for one-time proof actions beyond wallet linking.

**Public inputs:** commitment, nullifier, action_id

## IdentityVerifier

**Address:** `0xa2Cd20296027836dbD67874Ebd4a11Eeede292C8`

Verifier for the base identity circuit. Proves knowledge of the secret behind a commitment.

**Public inputs:** commitment

---

## IdentityVerifierRouter

**Address:** `0xC2D5F1C13e2603624a717409356DD48253f17319`

Routes identity verification requests to the correct verifier based on proof type. Allows the `IdentityRegistry` to support multiple proof formats without redeployment.

**Proof type registry:**

| Type ID | Circuit | Verifier |
|---|---|---|
| 0 | identity | IdentityVerifier |
| 1 | wallet_link | WalletLinkVerifier |
| 2 | nullifier | NullifierVerifier |

## IdentityBroadcaster

**Address:** `0x9A5710098B845e7841E1D09E1bde0dC1e30374AC`

Handles XCM message construction and dispatch for broadcasting identity and reputation state to other Polkadot parachains. See [XCM Integration](../xcm-integration/overview.md) for the full technical reference.

**Key functions:**

| Function | Description |
|---|---|
| `broadcastIdentity(bytes32 commitment, uint32 destParachain)` | Broadcast identity state via XCM |
| `broadcastReputation(address wallet, uint32 destParachain)` | Broadcast reputation state via XCM |
| `broadcastWalletLink(address wallet, bytes32 commitment, uint32 destParachain)` | Broadcast wallet link event |
| `enableAutoBroadcast(uint32 destParachain)` | Enable auto-broadcast on score change |
| `setDestinationContract(uint32 parachain, address receiver)` | Admin: configure receiver |
